# Da.Signa — статическая верстка сайта

Адаптивная, кроссбраузерная статическая верстка корпоративного сайта медицинского
центра **Da.Signa** по макету Figma. HTML + CSS + ванильный JavaScript, без сборщиков
и зависимостей.

Страницы: `index.html` (главная), `about.html` (О центре), `services.html` (Услуги),
`service.html` (шаблон услуги), `doctors.html` (Врачи), `contacts.html` (Контакты),
`404.html` (страница не найдена). Общие шапка, футер, форма записи и мобильное меню.

## Запуск

Статика — нужен любой http-сервер из корня проекта:

```bash
python3 -m http.server 5500
# открыть http://localhost:5500
```

## Структура

```
index.html / about.html /      # страницы (семантическая разметка, без инлайн-стилей)
services.html / service.html / #
doctors.html / contacts.html / #
404.html                       #
assets/images/                 # изображения из Figma, оптимизированы в WebP
assets/logo/                   # вектор-логотип из Figma: logo-dark.svg (шапка),
│                              #   logo-light.svg (футер)
assets/vendor/                 # самохостинг Lenis (lenis.min.js + lenis.css), pinned 1.1.18
src/
├── styles/
│   └── main.css               # единый стиль-лист: токены → типографика →
│                              #   глобальная база → секции (header, hero, about,
│                              #   advantages, services, specialists, appointment,
│                              #   contacts, footer, widgets, logo)
└── lib/
    └── main.js                # Lenis smooth-scroll + плавная якорная навигация,
                               #   меню, табы/аккордеон услуг, карусель специалистов,
                               #   валидация формы + маска телефона, модалка, reveal
```

Логотип — точный вектор из Figma (контуры букв), а не воссозданный шрифтом, поэтому
совпадает с макетом 1:1. Ссылки на CSS/JS снабжены `?v=` для обхода кэша браузера.

Все стили — строго в `src/styles/main.css`. Инлайн-стили не используются.
Порядок секций внутри `main.css` отражает каскад; каждый блок помечен
комментарием-заголовком секции.

## Секции главной

Header → Hero → О компании → Почему выбирают нас → Наши услуги → Специалисты →
Запишитесь на прием → Наши контакты → Footer, плюс плавающий виджет WhatsApp.

## Что реализовано

- **Адаптив:** mobile / tablet / desktop, брейкпоинты 375 / 768 / 901 / 1024 / 1440.
- **Плавный скролл:** Lenis (самохостинг), плавная якорная навигация с учётом высоты
  липкой шапки; отключается при `prefers-reduced-motion` и ставится на паузу, пока
  открыты модалка или мобильное меню.
- **Интерактив:** мобильное меню-дровер, услуги (табы на десктопе → аккордеон на
  мобильном), лента специалистов (свободный скролл — drag / трекпад / свайп, без
  стрелок, как в макете), фильтр врачей по специальности и модалка карточки врача
  (с возвратом фокуса и ловушкой Tab), форма записи с валидацией, маской телефона
  `+7 (___) ___ ____` и модалкой подтверждения.
- **SEO:** семантические теги, `h1…h3`, meta description, Open Graph, JSON-LD
  (`MedicalClinic`), `lang="ru"`.
- **Производительность:** WebP-изображения с явными `width/height`, `loading="lazy"`
  ниже первого экрана, preload + `fetchpriority` для hero, анимации только на
  `transform/opacity`.
- **Доступность:** фокус-стили, `aria-*` для меню/табов/формы/модалки,
  поддержка `prefers-reduced-motion`.

## Важно перед публикацией (плейсхолдеры)

| Что | Где | Действие |
|-----|-----|----------|
| **Шрифт Futura PT** | `src/styles/main.css` (секция Typography) | Сейчас подставлен бесплатный **Jost** (геометрический аналог). Для лицензионного Futura PT положите woff2 в `assets/fonts/` и раскомментируйте `@font-face` — он уже первый в стеке. |
| **Номер WhatsApp** | `index.html` (`.wa-widget`, соц-иконки) | Заменить `77271234567` на реальный. |
| **Карта** | `index.html` (`.contacts__map`) | Сейчас статичная картинка-ссылка. Можно заменить на iframe 2GIS/Яндекс.Карт. |
| **Фото врачей** | `assets/images/doctor.webp` | В макете один плейсхолдер на всех. Заменить реальными фото. |
| **Отправка формы** | `src/lib/main.js` (`form submit`) | Сейчас имитация (reset + модалка). Подключить POST на бэкенд / WordPress REST. |
| **Телефоны/адреса** | `index.html` (контакты, футер) | Сверить с актуальными данными клиента. |

## Замена шрифта-подстановки

Основной шрифт макета — **Futura PT** (коммерческий, Adobe Fonts). В верстке он
заменён бесплатным **Jost** с Google Fonts. Стек `--font-sans` уже включает
`Futura PT` первым — достаточно подключить файлы лицензионного шрифта.

- Добавить `<picture>` с AVIF-фолбэком к WebP при необходимости.
